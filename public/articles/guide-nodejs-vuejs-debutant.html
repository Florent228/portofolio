01 —Comprendre l'écosystème

Avant de tout installer, prenons 5 minutes pour comprendre ce que font ces outils. Cela t'évitera beaucoup de confusion.

Node.js — Le moteur JavaScript côté serveur

JavaScript a été inventé pour fonctionner dans les navigateurs (Chrome, Firefox…). Node.js permet de faire tourner JavaScript en dehors du navigateur, directement sur ton ordinateur ou sur un serveur.

Concrètement, Node.js te sert à :

  • Exécuter des scripts JavaScript sur ta machine
  • Faire tourner des outils de développement (comme Vue CLI)
  • Créer des serveurs web et des API

Analogie

Si JavaScript est une voiture, Node.js est la route qui lui permet de rouler en dehors de la ville (le navigateur).

npm — Le gestionnaire de paquets

Quand tu installes Node.js, tu obtiens automatiquement npm (Node Package Manager). C'est un outil en ligne de commande qui te permet de télécharger et gérer des librairies (des morceaux de code réutilisables créés par la communauté).

Analogie

npm, c'est comme un App Store pour les développeurs. Tu tapes une commande, et le code que tu veux est téléchargé et prêt à l'emploi.

Vue.js — Le framework front-end

Vue.js est un framework JavaScript qui facilite la création d'interfaces utilisateur (UI) dynamiques et réactives. Il est réputé pour sa courbe d'apprentissage douce, ce qui en fait un excellent choix pour les débutants.

Avec Vue.js, tu vas construire des composants : des blocs réutilisables qui combinent du HTML, du CSS et du JavaScript.

Le terminal (ou invite de commandes)

Tu vas beaucoup utiliser le terminal (aussi appelé console ou invite de commandes). C'est une interface en mode texte où tu tapes des commandes pour interagir avec ton ordinateur. Pas de panique, on verra exactement quoi taper à chaque étape.

02 —Installer Node.js

Étape 1 : Télécharger Node.js

Rends-toi sur nodejs.org. Tu verras deux versions disponibles :

  • LTS (Long Term Support) — Choisis celle-là. C'est la version stable et recommandée.
  • Current — La version la plus récente, mais moins stable.

Clique sur le bouton LTS pour télécharger l'installeur Windows (.msi).

Étape 2 : Lancer l'installation

  1. Double-clique sur le fichier .msi téléchargé.
  2. Accepte les termes de la licence.
  3. Laisse le chemin d'installation par défaut (C:\Program Files\nodejs\).
  4. Sur l'écran "Custom Setup", laisse tout coché par défaut (notamment "Add to PATH").
  5. Clique sur Install, puis Finish.

Important

L'option "Add to PATH" permet à Windows de reconnaître les commandes node et npm depuis n'importe quel terminal. Si tu la décoches, rien ne fonctionnera.

Étape 3 : Vérifier l'installation

Appuie sur Windows + R, tape cmd, et appuie sur Entrée pour ouvrir l'invite de commandes :

// Terminal — Vérification

node --version v20.11.0 npm --version 10.2.4

Succès

Si tu vois des numéros de version, Node.js est correctement installé ! Si tu vois une erreur du type "node n'est pas reconnu...", redémarre ton ordinateur et réessaie.

03 —Comprendre npm

Tu vas utiliser npm constamment. Voici les commandes de base à connaître :

Commande Ce qu'elle fait
npm install Installe toutes les dépendances d'un projet existant
npm install nom-du-paquet Installe un paquet spécifique
npm install -g nom-du-paquet Installe un paquet globalement (disponible partout)
npm run dev Lance le serveur de développement
npm run build Compile le projet pour la production

Le fichier package.json

Chaque projet Node.js possède un fichier package.json. C'est la carte d'identité de ton projet : il liste les dépendances, les scripts disponibles, le nom et la version du projet.

// package.json — Exemple

{ "name": "mon-projet", "version": "1.0.0", "scripts": { "dev": "vite", "build": "vite build" }, "dependencies": { "vue": "^3.4.0" } }

Le dossier node_modules

Quand tu lances npm install, toutes les librairies sont téléchargées dans un dossier node_modules. Ce dossier peut être très volumineux (plusieurs centaines de Mo). Ne le modifie jamais manuellement et ne le copie pas : il se regénère toujours avec npm install.

04 —Installer un éditeur de code : VS Code

Visual Studio Code (VS Code) est l'éditeur de code le plus populaire au monde. Il est gratuit, léger, et dispose d'un écosystème d'extensions gigantesque.

Étape 1 : Télécharger et installer

Rends-toi sur code.visualstudio.com, clique sur Download for Windows et lance l'installeur.

Étape 2 : Options d'installation importantes

Durant l'installation, coche impérativement ces options :

  • Ajouter l'action "Ouvrir avec Code" au menu contextuel des fichiers
  • Ajouter l'action "Ouvrir avec Code" au menu contextuel des répertoires
  • Ajouter à PATH (permet de taper code . dans le terminal)

Étape 3 : Premier tour de VS Code

Astuce

Le terminal intégré de VS Code (raccourci Ctrl + `) te permet de taper des commandes sans quitter l'éditeur. Tu t'en serviras tout le temps.

05 —Configurer VS Code pour Vue.js

VS Code devient vraiment puissant avec les bonnes extensions. Pour installer une extension : clique sur l'icône Extensions dans la barre latérale (ou Ctrl + Shift + X).

1. Vue - Official (anciennement Volar)

C'est l'extension officielle de Vue.js. Elle apporte la coloration syntaxique des fichiers .vue, l'autocomplétion, la détection des erreurs en temps réel et la navigation entre composants.

2. ESLint

Analyse ton code et te signale les erreurs et les mauvaises pratiques pendant que tu écris. Indispensable pour écrire du code propre dès le début.

3. Prettier - Code formatter

Prettier formate automatiquement ton code pour qu'il soit toujours propre et cohérent. Pour activer le formatage à la sauvegarde : Ctrl + ,, puis coche "Editor: Format On Save".

4. GitLens

Améliore l'intégration Git dans VS Code : tu vois qui a modifié chaque ligne, l'historique des fichiers, etc.

5. Auto Rename Tag

Quand tu renommes une balise HTML ouvrante, la balise fermante est automatiquement renommée. Un gain de temps fou.

06 —Installer Git

Git est un outil de contrôle de version. Il te permet de sauvegarder l'historique de ton code, de revenir à une version précédente, et de collaborer avec d'autres développeurs. Même si tu travailles seul, tu dois connaître Git.

Installation et configuration

Rends-toi sur git-scm.com et télécharge l'installeur Windows. Choix recommandés :

  • Éditeur par défaut : sélectionne Visual Studio Code
  • PATH environment : "Git from the command line and also from 3rd-party software"
  • Line ending : laisse l'option par défaut

// Terminal — Configuration de ton identité Git

git config --global user.name "Ton Prénom Nom" git config --global user.email "ton@email.com" # Vérifier l'installation git --version git version 2.43.0.windows.1

07 —Créer ton premier projet Vue.js

L'heure est venue ! On utilise Vite (prononcé "vit"), l'outil de création de projet officiel et recommandé par Vue.js.

Étape 1 : Créer le projet

Ouvre le terminal intégré de VS Code (Ctrl + `), navigue vers ton dossier de travail puis lance la création :

// Terminal — Création du projet

cd C:\Users\TonNom\Documents npm create vue@latest

npm te pose une série de questions. Voici les choix recommandés pour un premier projet :

// Réponses recommandées

✔ Project name: … mon-premier-projet ✔ Add TypeScript? … No ✔ Add JSX Support? … No ✔ Add Vue Router? … Yes ← pour la navigation entre pages ✔ Add Pinia? … No ✔ Add Vitest? … No ✔ Add an End-to-End Testing Solution? … No ✔ Add ESLint? … Yes ← qualité du code ✔ Add Prettier? … Yes ← formatage automatique

Étape 2 : Installer et lancer

// Terminal

cd mon-premier-projet npm install npm run dev VITE v5.0.0 ready in 300 ms ➜ Local: http://localhost:5173/ ➜ Network: use --host to expose

Ouvre ton navigateur et va sur http://localhost:5173. Tu devrais voir la page d'accueil Vue.js !

Localhost

localhost signifie "mon propre ordinateur". Le port 5173 est celui utilisé par Vite par défaut. L'application ne tourne que sur ta machine, personne d'autre ne peut y accéder.

08 —Structure du projet expliquée

Ouvre le dossier du projet dans VS Code (code . dans le terminal). Voici ce que tu vois :

mon-premier-projet/ │ ├── node_modules/ ← Librairies installées (ne pas toucher) │ ├── public/ ← Fichiers statiques (favicon, images) │ └── favicon.ico │ ├── src/ ← TON CODE est ici ! │ │ │ ├── assets/ ← Images, fonts, CSS global │ │ └── main.css │ │ │ ├── components/ ← Composants Vue réutilisables │ │ └── HelloWorld.vue │ │ │ ├── router/ ← Configuration de la navigation │ │ └── index.js │ │ │ ├── views/ ← Les "pages" de ton application │ │ ├── HomeView.vue │ │ └── AboutView.vue │ │ │ ├── App.vue ← Le composant racine │ └── main.js ← Initialisation de l'app Vue │ ├── index.html ← La page HTML de base ├── package.json ← Infos du projet et dépendances ├── vite.config.js ← Configuration de Vite └── .eslintrc.cjs ← Configuration d'ESLint

Les fichiers .vue — Le coeur de Vue.js

Chaque fichier .vue est un composant. Il est divisé en 3 sections :

// Composant Vue — Structure de base

<template> <!-- Le HTML de ton composant --> <div class="hello"> <h1>{{ message }}</h1> <button @click="changerMessage">Clique ici</button> </div> </template> <script setup> // Le JavaScript de ton composant import { ref } from 'vue' const message = ref('Bonjour le monde !') function changerMessage() { message.value = 'Tu as cliqué !' } </script> <style scoped> /* Le CSS de ton composant (scoped = uniquement pour CE composant) */ .hello { text-align: center; color: #42b883; } </style>

Les trois grandes règles à retenir :

  • <template> : Ce que l'utilisateur voit (HTML).
  • <script setup> : La logique (données, fonctions).
  • <style scoped> : Le style visuel (CSS, uniquement pour ce composant).

09 —Lancer et modifier ton application

Modifier ta première page

Ouvre le fichier src/views/HomeView.vue et remplace le contenu :

// src/views/HomeView.vue

<template> <main> <h1>Mon premier projet Vue.js !</h1> <p>Je suis en train d'apprendre Vue.js. C'est super !</p> </main> </template>

Sauvegarde (Ctrl + S) et regarde ton navigateur : la page se met à jour automatiquement sans recharger. C'est le Hot Module Replacement (HMR) de Vite.

Créer ton premier composant

Crée un fichier src/components/MonBouton.vue :

// src/components/MonBouton.vue

<template> <button class="mon-bouton" @click="$emit('click')"> <slot>Cliquez ici</slot> </button> </template> <script setup> defineEmits(['click']) </script> <style scoped> .mon-bouton { background-color: #42b883; color: white; border: none; padding: 10px 20px; border-radius: 6px; font-size: 16px; cursor: pointer; } .mon-bouton:hover { background-color: #33a06f; } </style>

Puis utilise-le dans HomeView.vue :

// src/views/HomeView.vue — Avec composant

<template> <main> <h1>Mon premier projet Vue.js !</h1> <MonBouton @click="direBonjour">Dire bonjour</MonBouton> <p v-if="monMessage">{{ monMessage }}</p> </main> </template> <script setup> import { ref } from 'vue' import MonBouton from '@/components/MonBouton.vue' const monMessage = ref('') function direBonjour() { monMessage.value = 'Bonjour !' } </script>

Sauvegarde et teste dans le navigateur. Ton bouton personnalisé fonctionne !

11 —Résolution des erreurs courantes

"node" n'est pas reconnu comme commande interne

Cause & Solution

Node.js n'est pas dans le PATH de Windows. Désinstalle Node.js, réinstalle-le en cochant bien l'option "Add to PATH", puis redémarre l'ordinateur.

EACCES: permission denied

Cause & Solution

Problème de droits d'accès. Lance le terminal en tant qu'administrateur (clic droit sur CMD > Exécuter en tant qu'administrateur).

npm ERR! code ENOENT ou module introuvable

Cause & Solution

Tu n'es pas dans le bon dossier, ou les dépendances ne sont pas installées. Vérifie avec pwd, puis lance npm install.

Port déjà utilisé (EADDRINUSE)

Cause & Solution

Une autre instance du serveur tourne déjà. Ferme l'autre terminal ou utilise npm run dev -- --port 3000 pour changer de port.

Le navigateur affiche une page blanche

Cause & Solution

Erreur JavaScript dans le code. Ouvre les outils développeur (F12 > onglet Console) pour voir l'erreur en détail.

VS Code ne colore pas les fichiers .vue

Cause & Solution

L'extension Vue - Official n'est pas installée. Va dans les extensions (Ctrl + Shift + X), cherche "Vue - Official", installe-la et recharge VS Code.

12 —Et maintenant ?

La meilleure façon d'apprendre est de construire des projets qui t'intéressent vraiment. Commence petit, et augmente la complexité progressivement.

Niveau 1 — Les bases de Vue.js

  • Lis la documentation officielle Vue.js (elle est excellente et disponible en français)
  • Comprends les concepts clés : réactivité (ref, reactive), directives (v-if, v-for, v-model), props et émissions

Niveau 2 — Aller plus loin

  • Apprends Vue Router pour créer une navigation multi-pages
  • Découvre Pinia pour gérer l'état global de ton application
  • Utilise une bibliothèque UI comme Vuetify ou PrimeVue

Niveau 3 — Écosystème complet

  • Connecte ton front-end Vue.js à un back-end Node.js (API REST avec Express.js)
  • Apprends les bases de Git et crée un compte GitHub
  • Explore TypeScript pour un code plus robuste

Conseil final

Commence petit (une todo list, une météo app, un quiz), et augmente la complexité progressivement. L'essentiel est de pratiquer régulièrement plutôt que d'essayer tout comprendre d'un coup. Bon développement !
